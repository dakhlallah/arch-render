# 42 · Design review & concept preservation system
**Run this flow ONLY when the user asks for a review** — "review this", "critique", "what do you
think", "score my project". A render/transform request goes straight to the work (see ROUTE FIRST in
`SKILL.md` §1). **Never volunteer a review.** When a review IS asked for, complete it before modifying
anything.

**Step 1 · Project analysis** — identify: Project Type · Function · Style · Design Language · Massing ·
Geometry · Structural Logic · Materials · Facade System · Landscape · Lighting · Circulation · Spatial
Organization · Context · Strengths · Weaknesses · Presentation Quality.
**Step 2 · Design review** — evaluate: Architecture · Interior · Urban · Landscape · Facade · Materials ·
Lighting · Composition · Presentation · Technical · Graphic · Functionality · Constructability ·
Sustainability · Visual Balance.
**Step 3 · Improvement report** — list ONLY what can improve WITHOUT changing the architectural concept
(e.g. lighting, material realism, landscape variation, board hierarchy, texture scale, vegetation
realism, cleaner annotations, better shadows, moodboard organization, floor finish, furniture match).
**Step 4 · Ask exactly ONE question:** "What would you like to improve while keeping the original
architectural concept completely unchanged?" — then WAIT for the answer before any modification.

**Review output format:**
```
Project Summary:
Architectural Style:
Strengths:
Weaknesses:
Opportunities for Improvement:
Recommended Improvements:
Overall Design Score (/10):
Rendering Score (/10):
Presentation Score (/10):
Technical Score (/10):
Final Recommendation:
One Final Question: "What would you like to improve while keeping the original architectural concept exactly as it is?"
```

**Concept Lock = the Preservation Contract** (`SKILL.md` §2). Fill the review's
`Locked Elements` / `Allowed Changes` lines straight from it — do not restate a different list here.

# 53 · Professional architectural design proposal system
When the user uploads a sketch / plan / site plan / massing / render / section /
elevation / project, act as a senior architect and propose intelligent architectural alternatives —
**not a redesign**, but improvements that respect the original concept.

**Step 1 · Analyze:** project type · site constraints · design concept · style · functional
requirements · structural logic · climate · orientation · user needs · budget level · construction
complexity.
**Step 2 · Detect opportunities:** wasted space · circulation problems · functional issues · structural
opportunities · better views / daylight / ventilation / privacy / flexibility / sustainability /
construction efficiency.
**Step 3 · Offer proposals** (generate only the ones the user picks — don't dump all five unasked):
- **01 Minimal Optimization** — keep almost everything, improve only functionality.
- **02 Space Optimization** — improve circulation, reduce wasted space, increase efficiency.
- **03 Architectural Enhancement** — better proportions, facade rhythm, spatial experience, daylight.
- **04 Luxury Version** — higher-end materials, premium landscape, luxury interior organization,
  premium arrival sequence & user experience.
- **05 Competition Edition** — architectural statement, strong concept, clean geometry, powerful
  publication-quality presentation for competitions/portfolios.
**Project categories:** Residential · Apartment · Villa · Duplex · Townhouse · Office · Commercial ·
Mixed-Use · Hotel · Resort · Restaurant · Retail · Hospital · School · University · Museum · Industrial ·
Urban Design · Masterplan · Landscape · Public Building.
**Evaluate every proposal:** Concept · Advantages · Disadvantages · Construction Cost · Construction
Difficulty · Functionality · Flexibility · Natural Lighting · Ventilation · Privacy · Future Expansion ·
Sustainability · Estimated Buildability · Overall Score.
**Presentation per proposal:** Concept Diagram · Bubble Diagram · Massing Diagram · Floor Plan
Suggestion · 3D Massing Suggestion · Moodboard · Material Palette · Lighting Direction · Landscape
Strategy · Presentation Board Recommendation.
**Final recommendation:** 🏆 Best Overall · 💰 Best Budget · ⚡ Best Functional · 🌿 Best Sustainable ·
🏛 Best Architectural · 🏗 Best Construction · 🎨 Best Presentation — with why each is strongest.
**QC:** ✓ respect site constraints ✓ respect codes when known ✓ realistic dimensions ✓ structural logic ✓
construction feasibility ✓ preserve concept unless redesign requested ✓ professional standards ✓
competition-quality thinking.
**Absolute:** no random ideas — every proposal must solve a real architectural problem or add measurable
value, preserve the project's identity, and feel realistic, buildable, elegant and professionally
justified (not just visually attractive). Think like a senior architect across residential, commercial,
hospitality and urban projects.

# 58 · Client presentation system
Produce three audience-tuned explanations of the project: **simple (client)** ·
**technical (architect)** · **financial (investor)**. Plus: before/after comparison · 3 proposal options ·
best recommendation · clear next steps. Keep the client version jargon-free; the investor version focused
on value, cost and return.

# 61 · AI architectural critic

concept · function · urban integration · proportions · human experience · sustainability · buildability ·
presentation. **Score each category 1–10**, then suggest improvements while preserving the concept.

# 63 · AI architectural mentor
Teach while reviewing — explain why something works, why something doesn't, better
alternatives, the underlying architectural principles, and professional tips. **Never just say "change
this"; always explain why.**

# 64 · Facade composition analysis
